    Using Your Dark Account

    Hi! Welcome!

    You’re one of our first users, and we’re excited to have you. Dark is primarily supported for Chrome.

    If you need support, the Dark Community Slack is the best place for front-line support. We’re very responsive during the day, and often responsive at all hours. Depending on the issue, we may need you to use a workaround. If you have an urgent issue you can call Ellen (248-202-7698) or Paul (415-341-2432).

    README

    Dark is a radically different approach to software development. There are three big things you must consider when using Dark:

    1. Dark uses a set of structural backend components (API, Workers, Scheduled Jobs/Cron, internal tools, datastores). Read this first.
    2. Dark requires you to develop from incoming requests/traces (using your traces, live values, play buttons, and return values). Read this second.
    3. Working safely in Dark relies on feature flags (Coming soon).

    Editor

    • If you are struggling with the editor and haven’t read them yet, we strongly recommend the overview of backend components & trace driven development. The best place to start is writing line-by-line and checking the live value at each step.
    • The editor is expression based, and it will help to think of it that way vs. as text or lines.
    • The editor still has some quirks (we’re sorry!) please tell us about places where you feel like you can’t type and/or are fighting with the editor. The best place to report bugs or feedback is the Dark Community Slack.
    • Error Handling in Dark.
    • Refactoring code in Dark.

    Language

    • If you have not used a functional language before, functional aspects of Dark has an overview of the functional features in Dark (implicit returns, pipelines, pattern matching, and map instead of foreach).
    • The editor supports the language, so DB:: will bring up all datastore functions, List:: all list functions, etc. You’ll also see the doc strings in line. That said, a list of all expressions is available here.
    • Each expression in the language is versioned independently (DB::getAll_v3, for instance). You’ll always have access to the version you are using, and the latest version.

    Additional Information/Advanced Topics

    • Collaboration: Works like Google docs. If you need to share a project with another user, please request an organizational canvas via Slack.
    • Hosting Static Assets
    • CORS: if you’re interested in setting CORS, we have internal tools do that! We do want to do it in real-time with you to make sure it works. Drop us a note in Slack and we can get you set up.

    Known Issues

    • Large handlers cause performance problems.
    • Recursion does not work well yet.
    • Some quirks exist around international keyboards. Please report all instances of this in Slack. In the short-term, you may need to switch to an English keyboard.
